# テストケース管理アプリ 要件定義書

## 1. アプリケーション概要

アプリのテストケースを管理するWebアプリケーション。テストケースの登録・管理と、テスト実行プロジェクトの作成・実行結果の記録を一元管理できる。

---

## 2. 機能要件

### 2.1 テストケース管理

#### テストケース

| 項目 | 内容 |
|------|------|
| 概要 | テストケースの目的・内容を簡潔に記述するフィールド |
| 優先度 | High / Medium / Low などの優先度を付与できる |
| テスト種別 | 機能テスト / 回帰テスト / 性能テスト / セキュリティテスト など |
| 複数登録 | テストケースは複数件登録できる |

#### テストステップ

- テストケース内に複数のテストステップを登録できる
- 各テストステップには以下を記入できる
  - **概要**: ステップの操作内容を短く記述
  - **詳細**: 具体的な手順・入力値・確認内容などを詳細に記述

```
テストケース
├── 概要
├── 優先度
├── テスト種別
└── テストステップ[]
    ├── 概要
    └── 詳細
```

---

### 2.2 テスト実行プロジェクト管理

#### テスト実行プロジェクト

- テスト実行プロジェクトを新規作成できる
- 作成済みのテストケースの中から、実行対象とするケースを選択して紐付けられる

#### 実行結果の記録

- テスト実行プロジェクト内で、各テストケースのステップごとに実行結果を設定できる
- 実行結果の候補例: `未実施` / `合格` / `不合格` / `ブロック`

```
テスト実行プロジェクト
├── プロジェクト名
├── 対象テストケース[] (既存テストケースから選択)
│   └── テストステップ[]
│       ├── 概要（参照）
│       ├── 詳細（参照）
│       └── 実行結果
```

## 3. データモデル（概念）

### TestCase（テストケース）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子 |
| title | string | 概要 |
| priority | enum | High / Medium / Low |
| testType | enum | テスト種別 |
| steps | TestStep[] | テストステップ一覧 |
| createdAt | datetime | 作成日時 |
| updatedAt | datetime | 更新日時 |

### TestStep（テストステップ）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子 |
| order | number | ステップの順序 |
| summary | string | 概要 |
| detail | string | 詳細 |

### TestProject（テスト実行プロジェクト）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子 |
| name | string | プロジェクト名 |
| testCases | TestCaseRef[] | 対象テストケース |
| createdAt | datetime | 作成日時 |

### TestStepResult（ステップ実行結果）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子 |
| projectId | string | テスト実行プロジェクトID |
| testCaseId | string | テストケースID |
| stepId | string | テストステップID |
| result | enum | 未実施 / 合格 / 不合格 / ブロック |
| note | string | 備考（任意） |

## 4. 画面一覧（想定）

| 画面名 | パス | 説明 |
|---|---|---|
| テストケース一覧 | `/testcases` | 登録済みテストケースの一覧表示・新規作成 |
| テストケース詳細/編集 | `/testcases/[id]` | テストケースの詳細表示・編集・ステップ管理 |
| テスト実行プロジェクト一覧 | `/projects` | 実行プロジェクト一覧・新規作成 |
| テスト実行プロジェクト詳細 | `/projects/[id]` | テストケース選択・実行結果の記録 |

---

## 5. 非機能要件（暫定）

- フロントエンド: Next.js (TypeScript / Tailwind CSS / App Router)
- データストア: 未定（ローカルストレージ / SQLite / PostgreSQL など）
- 認証: 不要（シングルユーザー想定）

---

## 6. 未決事項・検討事項

- [ ] データの永続化方法（DBの選定）
- [ ] テストケースのバージョン管理が必要か
- [ ] テスト実行プロジェクト内でテストケースをコピーするか参照するか
- [ ] テスト種別・優先度の選択肢をカスタマイズ可能にするか
- [ ] 実行結果に添付ファイルやスクリーンショットを追加するか
